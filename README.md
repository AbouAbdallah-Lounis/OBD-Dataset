# README.md

## Diesel Vehicle OBD-II PID Dataset

### Overview

This dataset contains OBD-II Parameter ID (PID) measurements collected from real diesel vehicles using the **ANCEL ELITE 610** diagnostic scanner. The objective of the dataset is to support research and development in areas such as:

* Vehicle fault diagnosis
* Predictive maintenance
* Machine learning-based fault detection
* Anomaly detection
* Multi-label fault classification
* Automotive data analytics

Raw diagnostic data were initially recorded in the proprietary **CBF format** generated by the ANCEL device. A custom parsing and preprocessing script was subsequently used to extract and structure the measurements into a tabular **CSV dataset** suitable for data analysis and machine learning applications.

---

## Data Collection Procedure

1. Vehicle diagnostic data were acquired through the OBD-II interface using the ANCEL ELITE 610 scanner.
2. Data were recorded from multiple diesel-powered vehicles under real operating conditions.
3. The scanner exported the recordings as CBF files.
4. A custom conversion script extracted selected PID measurements and transformed them into a structured CSV format.
5. Diagnostic Trouble Codes (DTCs) reported by each vehicle were manually documented.

---

## Dataset Features

The dataset contains the following OBD-II parameters.

| PID      | Description                       | Unit                           |
| -------- | --------------------------------- | ------------------------------ |
| LOAD_PCT | Calculated Engine Load            | %                              |
| ECT      | Engine Coolant Temperature        | °C                             |
| MAP      | Intake Manifold Absolute Pressure | kPa                            |
| RPM      | Engine Speed                      | revolutions per minute         |
| VSS      | Vehicle Speed Sensor              | km/h                           |
| IAT      | Intake Air Temperature            | °C                             |
| MAF      | Mass Air Flow Rate                | g/s                            |
| FRP      | Fuel Rail Pressure                | kPa or MPa (vehicle dependent) |
| BARO     | Barometric Pressure               | kPa                            |
| VPWR     | Control Module Voltage            | V                              |
| AAT      | Ambient Air Temperature           | °C                             |

## Dataset Structure

Each row in the dataset corresponds to a timestamped OBD-II measurement sample acquired from a vehicle.

Example:

```csv
vehicle_id,LOAD_PCT,ECT,MAP,RPM,VSS,IAT,MAF,FRP,BARO,VPWR,AAT,label
PEUGEOT_EXPERT_2014,25.8,82,102,1950,65,31,28.5,42000,99,13.8,30,1
```

Where:

* `vehicle_id` uniquely identifies the vehicle.
* PID columns contain sensor measurements.
* `label` may be used to indicate healthy/faulty status depending on the specific experiment.

## Potential Applications

This dataset can be used for:

* Binary fault detection (healthy vs faulty)
* Multi-label fault diagnosis
* Diagnostic Trouble Code prediction
* Vehicle health monitoring
* Predictive maintenance systems
* Time-series classification
* Anomaly detection
* Feature importance analysis
* Explainable AI for automotive diagnostics

---

## Limitations

* The dataset contains only diesel vehicles.
* The number of vehicles is limited.
* Some DTC classes are represented by a single vehicle.
* Driving conditions, ambient conditions, and operating regimes may vary between recordings.
* Sensor availability and measurement resolution may differ across vehicle manufacturers.
